# Linear Regression Analysis with RΒ²

## Project Overview
This project demonstrates linear regression analysis using pure linear algebra (no sklearn), 
examining how error variance affects RΒ² performance.

## What This Project Shows

### 1. The Relationship Between Error Variance and RΒ²
**Simple Explanation**: Variance = how spread out points are from the average.
- Low variance β†’ points close to the line β†’ high RΒ²
- High variance β†’ points scattered everywhere β†’ low RΒ²

Think of it like shooting arrows at a target:
- All arrows near center = low variance
- Arrows scattered all over = high variance

# Feature Engineering: Creating 20 New Variables

## Variable Creation Strategy

We created 20 additional variables using **non-linear transformations** rather than simple linear combinations. Specifically: 5 squared terms (Xβ‚Β², Xβ‚‚Β², etc.), 5 square root terms (β|Xβ‚†|, β|Xβ‚‡|, etc.), and 10 interaction terms (Xβ‚Γ—Xβ‚‚, Xβ‚Γ—Xβ‚ƒ, etc.). These are all **non-linear** transformations because they cannot be written as simple weighted sums of the original variables.

This approach is superior to using linear combinations (like Xβ‚ + Xβ‚‚ or 2Γ—Xβ‚ - 3Γ—Xβ‚‚) because linear combinations add no new information - the regression model can already capture any linear relationship between variables automatically. Non-linear features, however, allow the model to capture curved relationships, polynomial patterns, and variable interactions that a purely linear model would miss.

The result is genuine feature engineering that expands the model's capability to fit complex patterns, which is why we sometimes see meaningful improvements in RΒ² when adding these 20 variables, especially when noise levels are low.
